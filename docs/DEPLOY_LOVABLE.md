# Publicação e sincronização (Lovable + GitHub)

## Fluxo correto
1. Faça commit e push na branch `main`.
2. No Lovable, confirme que o projeto está conectado ao repositório correto: `analistaericksilva/bibliaalpha`.
3. No Lovable, execute **Publish/Deploy** para atualizar a versão publicada.

## Regras importantes
- O Lovable sincroniza principalmente a branch padrão (`main`).
- Renomear, mover ou deletar repositório/owner pode quebrar a conexão.
- Se desconectar e reconectar, o vínculo pode mudar e exigir nova confirmação de projeto.

## Checklist rápido de diagnóstico
- [ ] O commit aparece no GitHub em `main`
- [ ] O projeto Lovable está conectado ao repo certo
- [ ] A publicação mais recente no Lovable foi executada sem erro
- [ ] O domínio final aponta para o deploy mais recente
